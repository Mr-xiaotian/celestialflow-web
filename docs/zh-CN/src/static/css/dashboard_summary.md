# dashboard_summary.css

> 📅 最后更新日期: 2026/05/23

负责仪表盘右下角“总体状态摘要”面板的样式定义，采用色彩鲜明的统计卡片展示全局运行指标。

## 布局结构 (`.summary-grid`)

- 使用 `grid` 布局，固定为两列。
- 每个网格单元包含一个 `.summary-item`。

## 统计卡片 (`.summary-item`)

每个统计项根据其语义分配了不同的主题色，包含背景色（`.summary-item`）和数值文字颜色（`.summary-value`）：

| 统计项 | 类名 | 主色调 | 语义 |
|--------|------|--------|------|
| **总成功任务** | `.success` | `Jade` (绿) | 任务顺利完成 |
| **总等待任务** | `.pending` | `Carbon` (灰) | 队列中待处理 |
| **总错误任务** | `.error` | `Crimson` (红) | 处理失败，需关注 |
| **总重复任务** | `.duplicate` | `Marigold` (橙) | 命中去重逻辑 |
| **活动节点** | `.nodes` | `Cornflower` (蓝) | 当前正在运行的 Stage 数 |
| **总剩余时间** | `.remain` | `Violet` (紫) | 全局进度预估 |

## 样式特性

- **视觉层级**: 数值采用 `2rem` 加粗字体，标签采用 `0.75rem` 灰色小字。
- **暗黑模式**: 自动切换背景色至深色调（如 `900` 系列），并将文字颜色切换至浅色调（如 `300` 系列），确保对比度。
- **交互暗示**: 
  - 当总错误数大于 0 时，`.summary-value.error-clickable` 会显示手型光标，引导用户点击跳转查看错误详情。
